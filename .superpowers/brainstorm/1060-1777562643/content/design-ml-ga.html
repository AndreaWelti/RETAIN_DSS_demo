Design — Sezione 3: Modelli ML + Ottimizzatore GA

Surrogate models per ciascuna rotta + NSGA-II multi-obiettivo

LAYER 2 — Surrogate Models (uno per rotta × per output)
Architettura
  • Algoritmo: Gradient Boosting (XGBoost/LightGBM) — default
  • Input: 8 variabili materiale + 6 parametri processo
  • Output: un modello per ciascun KPI fisico (purezza, resa, energia_el, energia_th...)
  • Totale modelli: ~5 per rotta × 3 rotte = 15 modelli
  • Training: 80/20 split, cross-validation 5-fold
  • Metriche: R², RMSE, feature importance
Perché surrogate model?
Il GA deve valutare migliaia di soluzioni candidate per generazione.
Chiamare una simulazione numerica ad ogni valutazione sarebbe troppo lento.
Il surrogate model è ~1000× più veloce e sufficientemente accurato per guidare l'ottimizzazione.
I KPI economici (opex, revenue, net_margin) sono calcolati analiticamente sopra le predizioni fisiche — nessun modello ML separato.
LAYER 3 — Algoritmo Genetico NSGA-II (libreria: pymoo)
Flusso di ottimizzazione
1 L'utente inserisce le proprietà del telo in ingresso
2 GA ottimizza i parametri di processo di ciascuna rotta separatamente (3 run paralleli)
3 Per ogni rotta → Pareto front sui 5 obiettivi
4 route_selector confronta i 3 Pareto front e identifica la rotta dominante
✓ Output: rotta raccomandata + parametri ottimali + Pareto front visualizzato
