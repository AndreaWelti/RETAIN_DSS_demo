Design — Sezione 1: Architettura generale

Struttura del pacchetto retain_dss e flusso dei dati

retain_dss/ — struttura a livelli
LAYER 1 — Dati
data/
generator.py  ← genera DB sintetico
schema.py    ← definisce variabili
loader.py    ← carica/salva CSV
LAYER 2 — Modelli ML
models/
trainer.py   ← addestra surrogate model
predictor.py ← inferenza rapida
evaluator.py ← metriche R², RMSE
LAYER 3 — Ottimizzatore
optimizer/
genetic.py   ← algoritmo genetico (NSGA-II)
objectives.py ← funzioni obiettivo
route_selector.py ← confronta 3 rotte
LAYER 4 — Interfacce
ui/
app.py      ← Streamlit (operatori)
notebooks/  ← Jupyter (ricercatori)
INPUT: proprietà telo in ingresso (PVC%, PET%, contaminanti, granulometria…)  →  GA ottimizza i 3 processi in parallelo usando il surrogate model  →  OUTPUT: rotta migliore + parametri di processo ottimizzati + Pareto front

Note di design

  • Il database sintetico è un CSV (o SQLite) generato una sola volta e riusato da tutti i layer
  • I surrogate model (es. Random Forest o Gradient Boosting) sostituiscono le equazioni di processo durante l'ottimizzazione GA — molto più veloci da valutare
  • NSGA-II gestisce i 4 obiettivi (purezza PVC, resa, qualità, efficienza) restituendo un Pareto front
  • La UI Streamlit mostra slider per le proprietà di input e visualizza il Pareto front + la soluzione raccomandata
